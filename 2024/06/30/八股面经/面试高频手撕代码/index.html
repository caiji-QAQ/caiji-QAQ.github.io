1.给你两个绝对路径地址，然后你去把file1中的内容拷贝到file2中。

ofstream：用于文件的写入：

  • 1.创建ofstream对象：

      ofstream outputFile;

  • 2.打开文件并写入内容：

      outputFile.open("example.txt"); // 打开名为example.txt的文件进行写入

  if (outputFile.is_open()) {
      outputFile << "Hello, World!" << std::endl; // 将字符串写入文件
      outputFile << "This is a sample text." << std::endl;
  } else {
      std::cout << "Failed to open the file." << std::endl; // 如果文件打开失败，输出错误信息
  }

  outputFile.close(); // 关闭文件流

ifstream：用于文件的读取：

  • 1.创建ifstream inputFile

  • 2.打开文件并读取内容：

      inputFile.open("example.txt"); // 打开名为example.txt的文件进行读取

  if (inputFile.is_open()) {
      std::string line;
      while (std::getline(inputFile, line)) {
          std::cout << line << std::endl; // 输出文件中的每一行内容
      }
  } else {
      std::cout << "Failed to open the file." << std::endl; // 如果文件无法打开，输出错误信息
  }

  inputFile.close(); // 关闭文件流

getline

  • 1.std::getline(输入流, 目标字符串);
    getline(std::cin, line):从输入中读取一行文本，同时将其存在line。

  • 2.从文件中读取一行

    while (std::getline(file, line)) {
  std::cout << line << std::endl;
}

  • 3.自定义分隔符

      std::getline(stream, word, ',')

代码：

#include <iostream>
#include <fstream>
#include <string>

int main() {
std::string file1Path, file2Path;

std::cout << "请输入第一个文件的绝对路径 (file1): ";
std::cin >> file1Path;

std::cout << "请输入第二个文件的绝对路径 (file2): ";
std::cin >> file2Path;

std::ifstream file1(file1Path);
if (!file1) {
    std::cerr << "无法打开文件: " << file1Path << std::endl;
    return 1;
}

std::ofstream file2(file2Path);
if (!file2) {
    std::cerr << "无法打开文件: " << file2Path << std::endl;
    return 1;
}

std::string line;
while (std::getline(file1, line)) {
    file2 << line << "\n";
}

file1.close();
file2.close();

std::cout << "文件拷贝成功！" << std::endl;

return 0;
}

2.memcpy复制

#include<iostream>
// 手撕手撕memcopy，memcpy是一个用于内存拷贝的标准库函数
using namespace std;

void myMemcpy(void* dst,void* src,size_t size){
    char* ptr1 = static_cast<char*>(dst);
    char* ptr2 = static_cast<char*>(src);
    for(int i = 0;i<size;i++){
        *(ptr1+i) = *(ptr2+i);
    }
}
int main(){
    int a[] = {1,2,3,4,5};
    int b[5];
    myMemcpy(b,a,sizeof(b));
    for(int i = 0 ;i<5;i++){
        cout<<b[i];
    }
}

注意的点：

  • 1.实现时要将void*转换为char*，因为void*指针无法直接使用，转换为char*可以逐字节来复制
  • 2.memcpy的第三个参数size要设置为sizeof(a)

3.生产者消费者模型

1.单生产者-单消费者模型

单生产者-单消费者模型中只有一个生产者和一个消费者，生产者不停地往产品库中放入产品，消费者则从产品库中取走产品，产品库容积有限制，只能容纳一定数目的产品。因此有以下两种情况。
两种情况：

  • 1.生产者生产速度过快，因此需要等待消费者取走产品后才能继续防止新的产品。

  • 2.消费者消费速度过快，产品库里已经没有多余的产品了，生产者放入一个新的产品，消费者才能继续工作

    #include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <queue>
#include <ctime>
#include <windows.h>

using namespace std;
int i = 0;
queue<int> que;
mutex mtx; 
condition_variable cv;
bool done = false;

void producer(){
    for(int j =0;j<100;j++){
        unique_lock<mutex> lock(mtx);
        i++;
        que.push(i);
        cout<<"producer:"<<i<<endl;
        cv.notify_all();
        Sleep(40);
    }
    unique_lock<mutex> lock(mtx);
    done = true;
    cv.notify_all();

}

void consumer(){
    while(true){
        std::unique_lock<std::mutex> lock(mtx); 
        cv.wait(lock,[&]()->bool{return !que.empty()||done;});

        if(!que.empty()){
            int tmp = que.front();
            que.pop();
            cout<< "consumer:"<<tmp<<endl;
            Sleep(40);
        }
        else if(done == true){
            break;
        }

    }
}

int main(){
    thread a1(producer);
    thread a2(consumer);
    a1.join();
    a2.join();
}

在这里正好来聊一下条件变量的使用：

2.条件变量

我们可以使用条件变量（condition_variable）实现多个线程间的同步操作；当条件不满足时，相关线程被一直阻塞，直到某种条件出现，这些线程才会被唤醒。为了防止竞争，条件变量的使用总是和一个互斥锁结合在一起；通常情况下这个锁是std::mutex，并且管理这个锁只能是 std::unique_lockstd::mutex RAII模板类。

条件变量是利用线程间共享的全局变量进行同步的一种机制，主要包括两个动作：

  • 一个线程因等待”条件变量的条件成立”而挂起；
  • 另外一个线程使”条件成立”，给出信号，从而唤醒被等待的线程。

一般以以下两个方法实现：

  • 等待条件成立使用的是condition_variable类成员wait，wait_for 或 wait_until。
  • 给出信号使用的是condition_variable类成员notify_one或者notify_all函数。

2.1 如何使用条件变量

在条件变量中只能使用std::unique_lock<std::mutex>

  • wait()成员函数：在满足后面pred这个条件之前都会一直等待。

      void wait(std::unique_lock<std::mutex>& lock);
  //Predicate 谓词函数，可以普通函数或者lambda表达式
  template<class Predicate>
  void wait(std::unique_lock<std::mutex>& lock, Predicate pred);

  • wait_for()成员函数：其实就是多了一个时间长短。

      template<class Rep, class Period, class Predicate>
  bool wait_for(std::unique_lock<std::mutex>& lock,
              const std::chrono::duration<Rep, Period>& rel_time,
              Predicate pred);

  • wait_until()成员函数：到某一个时间点便不再等待

  • notify_one()\notify_all()：手动通知，有些时候线程的反应会比较慢，条件满足有一段时间但是还没反应过来，所以手动来通知一下他们唤醒一下等待线程。

为什么单独使用互斥锁不能实现等待机制？
互斥锁只是用来保护临界区（即共享资源）的，并不能用于在某些条件不满足时阻塞线程。例如：

  • 生产者加锁写入队列：如果只使用互斥锁，生产者在加锁后，会直接向队列写入数据，写完后解锁。消费者无法得知队列中有新的数据，也不会被阻塞。
  • 消费者加锁读取队列：如果消费者在加锁后发现队列为空，消费者也无法得知什么时候队列会有新数据，因此不能正确地等待生产者将数据放入队列。
  • 忙等待与资源浪费：如果消费者线程在队列为空时获取了锁并尝试消费数据，它会发现队列为空，但它不能阻塞，只能打印 “Queue is empty, waiting…” 并释放锁。随后，它又会再次尝试获取锁。这样一来，消费者线程会浪费大量的 CPU 时间在无效的检查上。

std::unique_lock<std::mutex> lock(mtx); 和直接使用 mtx.lock() 的区别

  • 管理方式：
    std::unique_lock<std::mutex> lock(mtx)是一个 RAII（Resource Acquisition Is Initialization）风格的锁管理器，它在构造时自动锁定互斥锁，在析构时自动解锁。
    mutex则是手动锁定互斥锁后，就必须适当位置调用mtx.unlock()否则就会自动释放了。
  • 异常安全性：由于 std::unique_lock 是 RAII 的实现，它保证了在任何情况下都能正确地释放锁，包括当代码抛出异常时。因此，它提供了更高的异常安全性。使用 mtx.lock() 后，如果代码在 mtx.unlock() 之前抛出异常，那么锁将不会被释放，这可能导致死锁，影响程序的稳定性。

2.2 条件变量工作机制

cv.wait() 对应唤醒 cv.notify_all()
cv.wait(lock, predicate)：这个函数使线程进入等待状态，直到满足以下两个条件之一：

  • predicate（判断条件）为 true。cv.wait() 会先检查 predicate，如果条件为 true，则线程不进入等待，直接继续执行。
  • 线程被其他线程通过 notify_one 或 notify_all 唤醒。
    当线程进入等待状态时，它会释放关联的互斥锁 lock，并在被唤醒时重新尝试获取锁。只有当 predicate 为 true 时，线程才会继续执行。

notify_one() 和 notify_all() 的作用：
提醒检查条件：notify_one() 和 notify_all() 的作用是通知等待在条件变量上的线程，可能有条件已经改变。它们会使线程从等待状态中醒来，重新检查条件。即使条件不满足，notify_all 也会唤醒所有等待的线程，然而被唤醒的线程会在检查条件后继续等待。

2.3 伪唤醒

伪唤醒是条件变量的一个特殊问题，其他的线程中不考虑为伪唤醒这个问题
伪唤醒是多线程编程中的一个特殊情况，指的是线程在没有收到任何条件变量通知（notify_one 或 notify_all）的情况下被意外唤醒。这种现象在某些操作系统中可能发生，即使没有任何线程显式地唤醒当前线程，它也可能被唤醒。

伪唤醒的原因：

  • 硬件中断：有些操作系统的条件变量机制可能会因为底层硬件中断或信号处理而意外地唤醒等待中的线程。
  • 操作系统实现：某些操作系统（例如早期的 POSIX 线程库实现）中，条件变量可能会因为内部实现的复杂性而在没有显式 notify 的情况下随机唤醒线程。
  • 调度器行为：操作系统调度器在某些极端情况下可能会错误地唤醒等待的线程，尽管这种情况极其罕见。

3.单生产者-多消费者模型

和单生产者-单消费者模型基本一样，只是在main函数换了个vector用于管理consumer

int main() {
    thread a1(producer);

    // 创建多个消费者线程
    vector<thread> consumers;
    for (int i = 0; i < 3; ++i) {  // 假设我们有3个消费者
        consumers.push_back(thread(consumer, i + 1));
    }

    a1.join();
    for (auto& c : consumers) {
        c.join();
    }

    return 0;
}

为什么出现了生产者先生产一批消费者再生产一批的情况呢？

  1. 生产者和消费者的执行时间差异：
    生产者和消费者的执行速度不同。可能生产者在多个连续的时间片中得到了执行，导致它可以连续生产多个元素，而消费者线程在此期间没有获得足够的 CPU 时间片来消费队列中的数据。
    在你提供的代码中，生产者在每次生产一个元素后都有 40 毫秒的延迟，消费者在消费一个元素后也有 40 毫秒的延迟。在操作系统的调度机制下，可能会连续调度生产者一段时间，然后才调度消费者，这就导致了生产一批再消费一批的情况。
  2. 线程调度机制：
    操作系统的线程调度机制可能导致生产者或消费者线程连续执行多个时间片。在多线程环境下，线程的调度取决于操作系统的策略，例如时间片轮转（Round-Robin）、优先级调度等。某个线程可能连续执行多个时间片，从而完成一系列生产或消费任务，而其他线程暂时被调度器挂起。
    如果操作系统优先调度生产者线程，那么生产者就有机会连续多次生产元素，然后消费者才被唤醒去处理。
