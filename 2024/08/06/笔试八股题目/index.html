2


解：
A. 成员个数：一个类的成员变量（包括数据成员和对象成员）的数量直接影响类的大小。成员变量越多，类的实例占用的内存就越大。
B. 函数个数：成员函数的数量本身并不直接影响类的实例大小，因为成员函数存储在代码段中，==不会占用对象的内存==。
C. 是否有虚函数：虚函数会影响类的大小，因为含有虚函数的类需要一个虚函数表指针（vptr）来指向虚函数表（vtable），这个指针会占用类实例的内存。
D. 虚函数个数：虚函数的数量会影响虚函数表的大小，但不会直接增加每个类实例的大小，只是增加了vtable的大小。
E. 静态变量个数：静态变量是类级别的，不是对象级别的，所以静态变量个数不会影响单个类实例的大小。
F. 对齐方式：内存对齐是指为了提高内存访问效率，编译器会在类成员之间插入适当的填充字节，使得成员变量在内存中按照特定的边界对齐，这会影响类的大小。

解：
这道题引出的知识点，什么情况初始化会是0什么情况初始化为随机值。

  • 1.全局和静态变量：全局和静态变量在程序启动时分配内存，并在 BSS 段（未初始化数据段）或数据段中分配。默认初始化为0
  • 2.局部变量数组：对于这些数组，如果没有显式初始化，它们的元素是未初始化的，其内容是未定义的（通常为内存中的随机值）。这意味着它们可能包含在该内存位置之前存储的任何数据。
  • 3.显式初始化：无论数组是全局、静态还是局部，如果显式初始化数组，则未初始化的元素将被默认初始化为零。

解：
经典的函数和指针传递问题，但是一直都没太搞懂==，这次争取给搞明白，传递的其实还是s的形参，并没有影响到main中s的值，main中的s仍然还是指向a，既然如此应该怎么才能做到从函数中来影响main中传递至此的值呢？

#include <stdio.h>
#include <stdlib.h>

void fun(int *p1, int *p2, int **s) {
    *s = (int *)malloc(sizeof(int)); // 修改实参s的值
    **s = *p1 + *(p2++); // 修改实参s所指向的值
}

int main() {
    int a[2] = {1, 2}, b[2] = {10, 20}, *s = a;
    fun(a, b, &s); // 传引用调用，传递的是s的地址
    printf("%d\n", *s); // 输出11，因为s现在指向的是分配的新内存
    free(s); // 释放分配的内存
    return 0;
}

也就是说原来的s传递相当于是形参传递，但是如果传递过来地址的话再用*s这样就可以对地址对应的值改变了，如果把题中的s=blabla删掉其实也是可以正常加和的。
